Pokemon-Monster

Web-design Project

This project is a Node.js/Express application that integrates a MySQL database to manage and display Pokémon-like monsters. ... The database consists of two tables: elements (storing element types such as fire, water, and grass) and monsters (containing monster data). The application features three main pages: Home, which displays the top five monsters by score and allows users to view detailed information via a modal; Monster List, which provides a form to add new monsters and a list of all monsters with options to update their scores; and Quiz, which presents a random monster with a multiple-choice quiz to guess its element type, graded through a secure Web API. The project also includes structured setup with EJS templates, public directories, and monster images for an engaging and interactive user experience.

EcoTrack

Android App Project

EcoTrack is an innovative environmental sustainability app I developed using Android Studio with IntelliJ IDEA. ... Designed to empower users in reducing their ecological footprint, EcoTrack combines user-friendly functionality with impactful features. It leverages the Room database framework to ensure secure storage of user data and offers capabilities such as user registration, login, and a unique username check. With EcoTrack, users can view their carbon footprint, track sustainable habits, discover eco-friendly products, participate in community challenges, and receive updates on environmental news. The app also enables setting eco-goals, attending workshops, engaging in environmental advocacy, and even creating personalized challenges. Additionally, EcoTrack includes a question-and-answer forum, trending topic suggestions, and options to change user information—making it a comprehensive platform for promoting eco-conscious living.

DrugStore Data System

Database/MongoDB

This project is a DrugStore Data System designed to manage prescriptions, patient data, and doctor profiles. ... It provides functionality for doctors to write new prescriptions, patients to request prescription fulfillment, and for users to register as new patients or doctors. The system also allows displaying both patient and doctor data. The back-end leverages Java for server-side processing and integration with MongoDB, a NoSQL database, to store and manage the application data efficiently. The user interface is simple and intuitive, facilitating easy navigation and usage for doctors and patients alike.
